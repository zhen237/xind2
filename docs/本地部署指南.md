# 本地部署指南

本文档描述如何在本地环境中部署和运行通信基建数智化平台，无需使用 Docker。

---

## 环境要求

### 必须安装的软件

| 软件 | 版本要求 | 用途 | 下载地址 |
| :--- | :--- | :--- | :--- |
| JDK | 21+ | 运行 Spring Boot 后端服务 | [Adoptium Temurin 21](https://adoptium.net/temurin/releases/?version=21) |
| Maven | 3.9+ | 构建后端项目 | [Apache Maven](https://maven.apache.org/download.cgi) |
| MySQL | 8.0+ | 数据库服务 | [MySQL Community Server](https://dev.mysql.com/downloads/mysql/) |
| Redis | 7.0+ | 缓存服务 | [Redis Windows](https://github.com/redis-windows/redis-windows/releases) |
| Node.js | 18+ | 运行前端项目 | [Node.js](https://nodejs.org/) |

### 环境变量配置（Windows）

1. **JAVA_HOME**: `C:\Program Files\Java\jdk-xx.x.x`
2. **MAVEN_HOME**: `C:\Program Files\Apache\maven-3.x.x`
3. 将 `%JAVA_HOME%\bin` 和 `%MAVEN_HOME%\bin` 添加到 `PATH`

### 验证安装

```powershell
# 验证 Java
java -version

# 验证 Maven
mvn -v

# 验证 Node.js
node -v
npm -v

# 验证 MySQL（启动后）
mysql --version

# 验证 Redis（启动后）
redis-cli ping
```

## 部署步骤

### 步骤 1：克隆仓库

```bash
git clone https://github.com/zhen237/xind2.git
cd xind2
```

### 步骤 2：配置数据库

#### 2.1 使用 Navicat 创建数据库（推荐）

**步骤 1：打开 Navicat 并连接 MySQL**

1. 双击打开 **Navicat Premium 16**
2. 在左侧面板，找到你的 MySQL 连接（如果没有连接，需要先创建）：
   - 点击左上角 **⚡ 连接** → 选择 **MySQL**
   - 填写连接信息：
     - **连接名**：随便取一个名字（如 `localhost`）
     - **主机**：`localhost` 或 `127.0.0.1`
     - **端口**：`3306`
     - **用户名**：`root`
     - **密码**：你的 MySQL 密码
   - 点击 **测试连接**，成功后点击 **确定**

**步骤 2：创建数据库**

1. 在左侧面板，右键点击你的 MySQL 连接（如 `localhost`）
2. 选择 **新建数据库** 📊
3. 在弹出的窗口中填写：
   - **数据库名**：`comm_platform`
   - **字符集**：选择 `utf8mb4`
   - **排序规则**：选择 `utf8mb4_unicode_ci`
4. 点击 **确定**

**步骤 3：导入初始化数据**

1. 在左侧面板找到刚创建的 `comm_platform` 数据库
2. **双击打开**数据库（图标会变成打开状态）
3. **右键点击** `comm_platform`
4. 选择 **运行 SQL 文件** 📄
5. 在弹出窗口中：
   - 点击 **...** 按钮，浏览并选择项目中的 `scripts/init-db.sql` 文件
   - **编码**：选择 `UTF-8`
   - 点击 **开始**
6. 等待执行完成，会显示 "执行成功"

**步骤 4：验证数据**

1. 展开 `comm_platform` → **表**
2. 双击 `m01_user` 表，确认有用户数据：
   - `admin` - 超级管理员
   - `operator` - 运维人员
   - `designer` - 设计人员

#### 2.2 使用命令行创建数据库（备选）

```bash
# 登录 MySQL
mysql -uroot -p

# 创建数据库
CREATE DATABASE comm_platform CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 导入初始化数据
USE comm_platform;
SOURCE scripts/init-db.sql;
```

#### 2.3 配置环境变量（推荐方式）

在项目根目录创建 `.env` 文件：

```env
# 数据库配置
MYSQL_URL=jdbc:mysql://localhost:3306/comm_platform?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
MYSQL_USER=root
MYSQL_PASSWORD=你的数据库密码

# JWT配置
JWT_SECRET=your-strong-secret-change-in-production
JWT_EXPIRATION=604800000
```

### 步骤 3：启动基础服务

#### 3.1 启动 Redis

```bash
# Windows - 进入 Redis 安装目录
cd D:\redis\Redis-7.2.14-Windows-x64-msys2
redis-server.exe redis.conf
```

#### 3.2 确保 MySQL 服务运行

- **Windows**: 打开服务管理器 (`services.msc`)，确保 MySQL 服务已启动
- **Linux/Mac**: `systemctl start mysqld`

### 步骤 4：启动后端服务

**启动 M01 认证服务**：

```bash
cd packages/m01-auth/backend
mvn spring-boot:run
```

- 服务地址：`http://localhost:8080`
- API 前缀：`/api/m01/`

**启动 M05 运维服务**（需要新开一个终端）：

```bash
cd packages/m05-twin-ops/backend
mvn spring-boot:run
```

- 服务地址：`http://localhost:8085`
- API 前缀：`/api/m05/`

### 步骤 5：启动前端服务

```bash
cd packages/m06-portal
npm install   # 首次运行需要安装依赖
npm run dev
```

- 访问地址：`http://localhost:5173`

## 常见问题

### Q: Maven 下载依赖慢怎么办？

**A:** 配置 Maven 国内镜像，修改 `~/.m2/settings.xml`：

```xml
<mirrors>
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### Q: 端口被占用怎么办？

**A:** 查找并释放端口：

```powershell
# Windows
netstat -ano | findstr :8080
taskkill /F /PID <进程ID>
```

### Q: 前端无法连接后端怎么办？

**A:** 检查：
1. 后端服务是否启动成功
2. 端口配置是否正确
3. 防火墙是否允许访问

### Q: 数据库连接失败？

**A:** 检查：
1. MySQL 服务是否运行
2. `.env` 文件中的数据库配置是否正确
3. 数据库密码是否正确
