# 本地部署指南

本文档描述如何在本地环境中部署和运行通信基建数智化平台，无需使用 Docker。

## 环境要求

### 必须安装的软件

| 软件 | 版本要求 | 用途 | 下载地址 |
| :--- | :--- | :--- | :--- |
| JDK | 17+ | 运行 Spring Boot 后端服务 | [Oracle JDK](https://www.oracle.com/java/technologies/downloads/#java17) |
| Maven | 3.8+ | 构建后端项目 | [Apache Maven](https://maven.apache.org/download.cgi) |
| MySQL | 8.0+ | 数据库服务 | [MySQL Community Server](https://dev.mysql.com/downloads/mysql/) |
| Redis | 7.0+ | 缓存服务 | [Redis for Windows](https://github.com/tporadowski/redis/releases) |
| Node.js | 18+ | 运行前端项目 | [Node.js](https://nodejs.org/) |

### 环境变量配置

#### Windows 系统

在系统环境变量中添加：

1. **JAVA_HOME**: `C:\Program Files\Java\jdk-17.x.x`
2. **MAVEN_HOME**: `C:\Program Files\Apache\maven-3.8.x`
3. 将 `%JAVA_HOME%\bin` 和 `%MAVEN_HOME%\bin` 添加到 `PATH`

#### 验证安装

```bash
# 验证 Java
java -version

# 验证 Maven
mvn -v

# 验证 Node.js
node -v
npm -v

# 验证 MySQL（启动后）
mysql --version

# 验证 Redis（启动后）
redis-cli ping
```

## 部署步骤

### 步骤 1：克隆仓库

```bash
git clone https://github.com/zhen237/xind2.git
cd xind2
```

### 步骤 2：配置数据库

#### 2.1 创建 MySQL 数据库

```bash
# 登录 MySQL
mysql -uroot -p

# 创建数据库
CREATE DATABASE example_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 2.2 初始化数据库表

```bash
mysql -uroot -proot123 example_db < scripts/init-db.sql
```

#### 2.3 配置后端服务

编辑各后端模块的配置文件：

**M01 认证服务** (`packages/m01-auth/backend/src/main/resources/application.yml`)：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/example_db?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root123
  redis:
    host: localhost
    port: 6379
server:
  port: 8080
```

**M05 运维服务** (`packages/m05-twin-ops/backend/src/main/resources/application.yml`)：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/example_db?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root123
  redis:
    host: localhost
    port: 6379
server:
  port: 8085
```

### 步骤 3：启动基础服务

#### 3.1 启动 Redis

```bash
# Windows
redis-server

# Linux/Mac
redis-server /etc/redis/redis.conf
```

#### 3.2 确保 MySQL 服务运行

```bash
# Windows - 检查服务状态
services.msc

# Linux/Mac
systemctl status mysqld
```

### 步骤 4：启动后端服务

**启动 M01 认证服务**：

```bash
cd packages/m01-auth/backend
mvn spring-boot:run
```

- 服务地址：`http://localhost:8080`
- API 前缀：`/api/m01/`

**启动 M05 运维服务**（需要新开一个终端）：

```bash
cd packages/m05-twin-ops/backend
mvn spring-boot:run
```

- 服务地址：`http://localhost:8085`
- API 前缀：`/api/m05/`

### 步骤 5：启动前端服务

```bash
cd packages/m06-portal
npm install   # 首次运行需要安装依赖
npm run dev
```

- 访问地址：`http://localhost:5173`
- 默认账号：`admin` / `admin123`

### 热更新

| 模块 | 热更新方式 |
| :--- | :--- |

| **前端** | Vite 已配置热更新，修改代码后自动刷新浏览器 |

## 常见问题

### Q: Maven 下载依赖慢怎么办？

**A:** 配置 Maven 国内镜像，修改 `~/.m2/settings.xml`：

```xml
<mirrors>
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### Q: 端口被占用怎么办？

**A:** 查找并释放端口：

```bash
# Windows
netstat -ano | findstr :8080
taskkill /F /PID <进程ID>

# Linux/Mac
lsof -i :8080
kill -9 <进程ID>
```

### Q: 前端无法连接后端怎么办？

**A:** 检查：
1. 后端服务是否启动成功
2. 端口配置是否正确
3. 防火墙是否允许访问
