# 通信基建数智化平台 - 项目分工与任务列表

---

## 一、项目概述

本项目采用**模块隔离架构**，将平台划分为多个独立模块，由不同人员负责开发。各模块通过 API 接口进行通信，实现松耦合协作。

## 四、基础功能通用要求

### 4.1 后端开发规范

| 要求 | 说明 |
| :--- | :--- |
| **技术栈** | Java 17 + Spring Boot 3.2.x + MyBatis Plus |
| **包路径** | `com.comm.mXX`（XX为模块编号） |
| **数据库表前缀** | `mXX_`（XX为模块编号） |
| **API 风格** | RESTful，统一响应格式 |
| **响应结构** | `{ "success": boolean, "code": int, "message": string, "data": object }` |
| **分页支持** | 支持 pageNum、pageSize 参数 |
| **错误处理** | 统一异常处理、日志记录 |

### 4.2 前端开发规范

| 要求 | 说明 |
| :--- | :--- |
| **技术栈** | Vue 3 + Vite + Element Plus |
| **路由配置** | 独立路由，通过 iframe 集成到 M06 |
| **代理配置** | vite.config.js 配置后端 API 代理 |
| **样式规范** | 遵循平台统一主题色（科技感深色主题） |
| **交互规范** | 统一的 loading、消息提示、错误处理 |

## 六、代码隔离规则

### 6.1 目录隔离

```
xind2/
├── packages/
│   ├── m01-auth/          ← 人员一专属
│   ├── m03-bim-gis/       ← 人员二专属
│   ├── m04-delivery/      ← 人员四专属
│   ├── m05-twin-ops/      ← 人员三专属
│   ├── m06-portal/        ← 门户（共享）
│   └── screen/            ← 人员五专属
│
├── scripts/               ← 共享脚本
├── docs/                  ← 共享文档
└── docker-compose.yml     ← 共享基础设施
```

### 6.2 隔离原则

- ✅ 只修改自己模块目录下的文件
- ✅ 通过 API 调用进行跨模块通信
- ❌ 禁止跨模块 import
- ❌ 禁止直接修改其他模块的数据库表
- ❌ 禁止修改共享文件前未与团队同步
