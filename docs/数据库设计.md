# 数据库设计

## 一、数据库概述

本系统采用MySQL 8.0作为主要数据存储，使用MyBatis-Plus作为ORM框架。数据库名为`comm_platform`。

### 1.1 连接信息

| 模块 | 用户名 | 密码 | 说明 |
|------|--------|------|------|
| M01 | root | Admin@123 | application.yml 中配置 |
| M03 | root | Admin@123 | application.yml 中硬编码 |
| M05 | appuser | apppass123 | application.yml 中配置 |

> **注意**：M03 的数据库密码在 `application.yml` 中硬编码，M05 使用独立的数据库用户 `appuser`。生产环境应统一使用环境变量管理敏感信息。

```sql
-- M03 设备表索引

```

## 六、注意事项

1. 所有表都应有创建时间和更新时间字段
2. 敏感数据（如密码）需要加密存储
3. 删除操作应使用逻辑删除而非物理删除
4. 定期备份重要数据
5. 大数据量表需要分区或分表处理
6. `m03_region` 表在代码中存在实体类但 `init-db.sql` 中缺少建表语句，部署时需手动补充
7. M05 模块使用独立数据库用户 `appuser`，与 M01/M03 的 `root` 用户不同
