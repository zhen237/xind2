# 通信基建数智化平台 — 五人AI协作分工方案

## 总览

五个人各自使用AI辅助开发，五个模块**代码层面完全独立**（独立的数据库表、独立的前端项目、独立的后端服务），仅通过以下方式松耦合：

- **JWT 密钥统一**：所有后端使用相同的 JWT secret 验证 token
- **共享只读表**：`shared_` 前缀的表，所有模块可读
- **M06 门户**：通过 iframe 加载各模块前端，菜单配置 iframeUrl
- **数据库同实例**：共用 MySQL `comm_platform`，通过表前缀隔离

## 人员四：M04 项目进程管理与视频监理

### 负责模块
`packages/m04-delivery/`（新建模块）

### 模块定位

### 当前状态
**全新模块**，后端和前端均需从零开发。

### 核心功能

#### A. 项目全流程管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **项目阶段跟踪** | 规划设计 → 三维设计 → 施工建设 → 竣工验收 → 运维移交，可视化进度 | P0 |
| **项目统计看板** | 项目总数、各阶段数量、完成率、延期项目、甘特图 | P0 |
| **施工任务分解** | WBS 任务分解、派发、进度上报 | P1 |
| **验收流程** | 分项验收/竣工验收模板，问题记录整改闭环，电子签章 | P1 |
| **工单管理** | 接收 M05 告警生成的工单，派发→处理→反馈→评价 | P0 |

#### B. 视频现场监理（五大检查项）

##### 检查项一：人证合一
- **要求**：由远及近拍摄持证人员及证件
- **验证内容**：
  - 人脸与证书照片比对（可先人工审核，后期接AI）
  - 证书编号、发证机关、有效期
  - 证书类型是否匹配岗位要求（电工证、登高证、安全员证等）
- **数据字段**：人员姓名、身份证号、证书类型、证书编号、发证机关、有效期起止、现场照片（远+近各一张）、视频片段路径、审核结果、审核人

##### 检查项二：个人安全防护
- **要求**：清晰拍出日期和有效期
- **检查内容**：
  - **安全帽**：生产日期、有效期（通常3年）、外观破损检查
  - **安全带**：生产日期、有效期、外观磨损检查
  - **防坠器**：数量核对、最近检测日期、检测报告编号
  - **绝缘鞋/手套**（外电作业时）：生产日期、有效期、绝缘等级
- **数据字段**：检查日期、设备类型、品牌/型号、生产日期、有效期、检测日期、外观状态、数量、照片（含日期特写）、检查结论

##### 检查项三：施工总体情况
- **要求**：现场负责人介绍项目名称、参建单位、地点、时间、内容；由远及近体现现场环境无隐患
- **录制内容**：
  - 项目名称、参建单位（建设/设计/施工/监理）
  - 施工地点、时间、当日工作内容
  - 现场环境全景（远处→近处）：地面平整、材料堆放、通道畅通
- **数据字段**：项目ID、负责人姓名、录制时间、视频路径、环境评估结论、隐患描述

##### 检查项四：安全围栏及警示标志
- **要求**：从路上拍摄围挡和标志，说明环境无隐患
- **检查内容**：
  - 围挡完整性（无缺口、高度达标）
  - 警示标志齐全（施工警示牌、安全标语、危险区域标识）
  - 夜间警示灯功能正常
  - 周围环境（道路、行人、建筑物）无安全隐患
- **数据字段**：检查日期、围挡状态、标志清单、照片（从道路角度）、检查结论、整改要求

##### 检查项五：临时用电
- **要求**：一机、一闸、一漏、一箱
- **检查内容**：
  - 配电箱固定牢固、接地保护
  - 漏电保护器功能测试（每个回路独立）
  - 一机一闸一漏一箱落实情况
  - 线缆连接（无破损、无裸露、架空/埋地规范）
  - 配电箱周围无杂物、有操作空间
- **数据字段**：检查日期、配电箱编号、回路数量、漏电保护器型号/数量、线缆状态、接地电阻值、照片、检查结论

## 统一开发规范（五人共同遵守）

### 1. 数据库
- 表前缀：`m01_`  `m03_`  `m04_`  `m05_`  `shared_`
- 共享只读表（shared_前缀）只能由 M01 或约定模块写入
- 每个模块在自己的 `packages/xxx/backend/src/main/resources/` 下维护增量 SQL 脚本

### 2. JWT 认证
- 统一密钥：环境变量 `JWT_SECRET`，默认值五人约定一致
- 统一过期时间：604800000ms（7天）
- 各模块后端包含 `JwtUtils` 类用于验证 token
- 前端 Axios 拦截器统一加 `Authorization: Bearer <token>`

### 3. API 规范
- 统一响应格式：`{ "code": 200, "message": "success", "data": {...} }`
- 统一错误码（见 api.md 错误码表）
- RESTful 风格：GET/POST/PUT/DELETE

### 4. 前端集成
- 各模块前端独立开发、独立部署
- 通过 M06 门户 iframe 加载
- 监听 `message` 事件接收 JWT Token：
  ```js
  window.addEventListener('message', (event) => {
    if (event.data.type === 'TOKEN') {
      localStorage.setItem('token', event.data.token)
    }
  })
  ```
