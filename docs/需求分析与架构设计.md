# 需求分析与架构设计

## 一、系统概述

通信基建数智化平台旨在为通信基建项目提供全流程数字管理解决方案，覆盖从规划设计、建设施工到运维监控的完整生命周期。

## 二、需求分析

### 2.1 功能需求

#### 2.1.1 用户管理模块（M01）
- 用户登录、Token验证
- 用户CRUD管理
- 角色权限管理
- 菜单动态配置
- 操作日志记录

#### 2.1.2 项目管理模块（M05）
- 项目基本信息管理
- 项目阶段进度跟踪
- 视频审核流程
- 告警事件处理

#### 2.1.3 BIM+GIS模块（M03）
- 3D地图展示（Cesium）
- 基站环境建模
- 天线设备管理
- 覆盖评估分析

#### 2.1.4 前端门户（M06）
- 统一登录入口
- 导航菜单管理
- 页面路由配置
- 全局状态管理

### 2.2 非功能需求

| 需求类型 | 具体要求 |
|----------|----------|
| 性能 | 接口响应时间 < 200ms |
| 可用性 | 系统可用性 > 99.9% |
| 安全性 | JWT认证，数据传输加密 |
| 可扩展性 | 微服务架构，支持水平扩展 |

## 三、架构设计

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    M06 统一前端门户 (Vue3)                       │
│  - 登录页 / 动态菜单 / iframe 容器                               │
└───────────────┬─────────────────────────────────────────────────┘
                │ iframe + postMessage
    ┌───────────┼───────────┬───────────┬───────────┐
    ▼           ▼           ▼           ▼           ▼
┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌─────────┐
│ M02   │  │ M03   │  │ M04   │  │ M05   │  │ 大屏中心 │
│网络规划│  │BIM+GIS│  │数智交付│  │数字孪生│  │  (Vue)  │
└───┬───┘  └───┬───┘  └───┬───┘  └───┬───┘  └────┬────┘
    │          │          │          │           │
    └──────────┴──────────┴──────────┴───────────┘
                    │ 直连共用数据库
            ┌───────┴─────────────────┬──────────────┐
            ▼                         ▼              ▼
    ┌───────────────┐          ┌─────────────┐  ┌────────┐
    │ MySQL + Redis │          │   InfluxDB  │  │ MinIO  │
    └───────────────┘          └─────────────┘  └────────┘

┌─────────────────────────────────────────────────────┐
│                    M01 统一认证服务                    │
│          - JWT签发 / 用户管理 / 菜单管理              │
└─────────────────────────────────────────────────────┘
```

### 3.2 技术选型

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | Vue3 | ^3.4.21 | 组合式API |
| UI组件库 | Element Plus | ^2.6.3 | Vue3组件库 |
| 3D引擎 | CesiumJS | ^1.141.0 | 地理可视化 |
| 后端框架 | Spring Boot | 3.1.10 | Java生态 |
| ORM框架 | MyBatis-Plus | 3.5.5 | 增强MyBatis |
| 数据库 | MySQL | 8.0.46 | 关系型数据库 |

### 3.3 安全设计

1. **认证方式**：JWT无状态令牌
2. **授权模型**：RBAC基于角色的访问控制
3. **会话管理**：Token黑名单机制
4. **数据安全**：敏感数据加密存储

## 五、数据模型

### 5.1 核心实体

- **用户(m01_user)**：id, username, password, real_name, email, phone, status
- **角色(m01_role)**：id, role_code, role_name, description
- **菜单(m01_menu)**：id, parent_id, menu_code, menu_name, menu_type, iframe_url, permission_code
- **M03项目(m03_project)**：id, project_name, project_code, region_code, status, creator_id
- **M03设备(m03_device)**：id, device_code, device_name, device_type, station_code, longitude, latitude, height, status
- **M03模型(m03_model)**：id, model_name, model_code, model_type, file_path, scale
- **M05设备(m05_device)**：id, device_code, device_name, device_type, station_code, status, manufacturer, model
- **M05告警(m05_alert)**：id, device_id, device_code, alert_content, level, status, source, order_no

### 5.2 关联关系

```
m01_user ──N:N──> m01_role ──N:N──> m01_menu

m03_project ──1:N──> m03_device
m05_device  ──1:N──> m05_alert
```
