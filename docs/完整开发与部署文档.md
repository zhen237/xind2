## 1. 整体架构

### 1.1 架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                    M06 统一前端门户 (Vue3)                       │
│  - 登录页 / 动态菜单 / iframe 容器 / 个人中心                     │
└───────────────┬─────────────────────────────────────────────────┘
                │ iframe + postMessage (传递 JWT)
    ┌───────────┼───────────┬───────────┬───────────┐
    ▼           ▼           ▼           ▼           ▼
┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌─────────┐
│ M02   │  │ M03   │  │ M04   │  │ M05   │  │ 大屏中心 │
│Vue+Java│  │Vue+Java│  │Vue+Java│  │Vue+Java│  │ (Vue)   │
└───┬───┘  └───┬───┘  └───┬───┘  └───┬───┘  └────┬────┘
    │          │          │          │           │
    └──────────┴──────────┴──────────┴───────────┘
                    │ 直接读写共用数据库/存储
            ┌───────┴─────────────────┬──────────────┐
            ▼                         ▼              ▼
    ┌───────────────┐          ┌─────────────┐  ┌────────┐
    │   MySQL 8.0   │          │   InfluxDB  │  │ MinIO  │
    │   (共用)      │          │   (共用)    │  │ (共用) │
    └───────────────┘          └─────────────┘  └────────┘
```

### 1.2 核心设计原则

- **模块解耦**：M02~M05 业务模块前后端各自独立，**禁止模块间直接 API 调用**。数据共享通过数据库表（只读）或 MinIO 文件实现。
- **统一认证**：M01 仅负责登录、JWT 签发、用户信息、菜单树。其他模块后端使用相同 JWT 密钥自行验证 token。
- **统一门户**：M06 为纯前端壳，动态加载 iframe 子应用。菜单数据来自 M01，权限控制在前端菜单显示。
- **数据库共用**：所有模块直连同一 MySQL 8.0 实例，通过表名前缀隔离写操作，共享只读表使用 `shared_` 前缀。
- **在线编译器友好**：所有配置通过环境变量注入，支持 GitHub Codespaces / GitPod 一键启动。

### 7.3 JWT 工具类

```java
@Component
public class JwtUtils {
    @Value("${jwt.secret}")
    private String secret;
    @Value("${jwt.expiration}")
    private Long expiration;

    public String generateToken(Long userId, String username) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + expiration);
        return Jwts.builder()
                .setSubject(userId.toString())
                .claim("username", username)
                .setIssuedAt(now)
                .setExpiration(expiryDate)
                .signWith(SignatureAlgorithm.HS512, secret)
                .compact();
    }

    public Long getUserIdFromToken(String token) {
        Claims claims = Jwts.parser().setSigningKey(secret).parseClaimsJws(token).getBody();
        return Long.parseLong(claims.getSubject());
    }

    public boolean validateToken(String token) {
        try {
            Jwts.parser().setSigningKey(secret).parseClaimsJws(token);
            return true;
        } catch (JwtException | IllegalArgumentException e) {
            return false;
        }
    }
}
```

### 7.4 登录接口

```java
@RestController
@RequestMapping("/api/m01/auth")
public class AuthController {
    @Autowired private UserService userService;
    @Autowired private JwtUtils jwtUtils;

    @PostMapping("/login")
    public ResponseEntity<?> login(@RequestBody LoginRequest request) {
        User user = userService.findByUsername(request.getUsername());
        if (user == null || !passwordEncoder.matches(request.getPassword(), user.getPassword())) {
            return ResponseEntity.status(401).body("用户名或密码错误");
        }
        String token = jwtUtils.generateToken(user.getId(), user.getUsername());
        return ResponseEntity.ok(new LoginResponse(token, user.getId(), user.getUsername()));
    }
}
```

#### 8.1.2 JWT 验证过滤器（每个业务模块必须）

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    @Autowired private JwtUtils jwtUtils;  // 与 M01 的 JwtUtils 逻辑相同

    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response,
                                    FilterChain chain) throws ServletException, IOException {
        String authHeader = request.getHeader("Authorization");
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);
            if (jwtUtils.validateToken(token)) {
                Long userId = jwtUtils.getUserIdFromToken(token);
                // 可存入 SecurityContext，也可不存，只做身份识别
                request.setAttribute("userId", userId);
            } else {
                response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
                return;
            }
        } else {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            return;
        }
        chain.doFilter(request, response);
    }
}
```

> 注意：各模块的 `JwtUtils` 必须使用相同的 secret 和算法。

#### 9.2.2 布局组件（App.vue）

```vue
<template>
  <el-container v-if="isLoggedIn" style="height: 100vh">
    <el-header>...</el-header>
    <el-container>
      <el-aside width="200px">
        <el-menu :default-active="activeMenu" @select="handleMenuSelect">

            <el-sub-menu v-if="item.children" :index="item.menuCode">
              <template #title>{{ item.menuName }}</template>

              </el-menu-item>

            </el-menu-item>

const menus = computed(() => userStore.menus)

const iframeRef = ref(null)

function handleMenuSelect(menuCode) {
  const menu = findMenuByCode(menus.value, menuCode)
  if (menu && menu.iframeUrl) {
    currentUrl.value = menu.iframeUrl
  }
}

function findMenuByCode(menuList, code) {
  for (const m of menuList) {
    if (m.menuCode === code) return m
    if (m.children) {
      const found = findMenuByCode(m.children, code)
      if (found) return found
    }
  }
  return null
}

// 向 iframe 传递 token
watch(currentUrl, (url) => {
  if (iframeRef.value && url) {
    iframeRef.value.contentWindow.postMessage({ type: 'TOKEN', token: userStore.token }, '*')
  }
})

onMounted(() => {
  if (userStore.token) {
    userStore.fetchMenus()
  }
})
</script>
```

## 10. 跨模块数据交换规范

### 10.1 原则

- **禁止模块间直接 REST API 调用**。
- **只读共享数据**：通过 `shared_` 前缀表，所有模块可 SELECT，但只能由 M01 或约定模块写入。
- **读写分离**：若 M05 需要 M03 生成的模型文件路径，M05 直接 SELECT `m03_model` 表即可（只读）。
- **工单流转**：M05 产生告警后，向 `m04_work_order` 表 INSERT 一条记录，M04 前端展示，无需 API 调用。

### 10.2 示例：M05 告警生成工单

```java
// 在 M05 的告警处理服务中
@Transactional
public void createWorkOrderFromAlert(Alert alert) {
    // 1. 插入告警记录到 m05_alert
    m05AlertMapper.insert(alert);
    // 2. 生成工单，直接插入 m04_work_order 表
    M04WorkOrder order = new M04WorkOrder();
    order.setOrderNo("WO" + System.currentTimeMillis());
    order.setTitle("告警：" + alert.getAlertContent());
    order.setStatus(0); // 待处理
    // 注意：这里使用 MyBatis Plus 的通用 Mapper，需要引入 M04 的实体类？
    // 解决方案：在 M05 项目中复制一份 M04WorkOrder 实体（只包含必要字段），使用不同的表名映射。
    // 或者使用 JDBC Template 直接插入。
    jdbcTemplate.update("INSERT INTO m04_work_order (order_no, title, status, create_time) VALUES (?, ?, ?, NOW())",
                        order.getOrderNo(), order.getTitle(), order.getStatus());
}
```

> 为避免依赖 M04 的实体 JAR，可使用原生 JDBC 或 MyBatis 的注解方式只映射必要字段。

### 10.3 共享只读表

例如 `shared_region` 区域表，由 M01 后台管理维护，其他模块可以查询用于下拉选择。

## 12. 常见问题

### Q1：在 Codespaces 中，其他模块的前端如何被 iframe 加载？

A：M06 门户的菜单表中配置的 `iframe_url` 可以使用 Codespaces 生成的转发域名，例如 `https://your-codespace-5174.preview.app.github.dev`。每个前端服务需要将 Vite 的 `server.host` 设置为 `0.0.0.0`，并启用 `server.hmr.clientPort` 等配置。

### Q2：各模块后端如何共享 JWT 工具类？

A：可以创建一个独立的 Maven 模块 `common-jwt`，打包成 JAR，然后各模块引入。初期直接复制代码也 OK，只要保证 secret 相同。

### Q3：MySQL 的空间查询如何在 MyBatis Plus 中使用？

A：定义实体中的 `geometry` 字段类型为 `String` 或使用 `com.baomidou.mybatisplus.extension.handlers.GeometryTypeHandler`。查询时使用自定义 SQL，调用 MySQL 空间函数，例如 `ST_Contains(geom1, geom2)`。

### Q4：多人同时开发，数据库表结构如何同步？

A：非共用表各自维护 SQL 文件，在 PR 描述中告知。共用表（如 `m01_*`、`shared_*`）由 M01 负责人统一修改，并通知所有人执行。
