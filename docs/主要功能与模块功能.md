# 通信基建数智化全流程平台 – 主要功能与模块功能清单

## 一、平台总体功能定位

本平台服务于通信基础设施的全生命周期管理，覆盖**规划设计 → 三维设计 → 交付验收 → 数字孪生运维**四大阶段。通过模块化、微前端架构，整合 GIS/BIM 可视化、5G 网络仿真、智能交付工作流、物联网实时监控与 AI 告警等能力，为运营商、设计院、施工单位、运维团队提供一站式数字化协同工具。

### 核心能力概览

| 能力域               | 核心价值                                                     |
| -------------------- | ------------------------------------------------------------ |
| **数通规划与仿真**   | 5G 基站站址智能规划、覆盖仿真、干扰分析，输出可落地方案      |
| **BIM+GIS 三维设计** | 基于真实地形与 BIM 模型的通信机房/基站/管线三维设计，碰撞检测，自动算量 |
| **数智化交付**       | 施工流程线上化、自动化生成图纸/料单、竣工验收数字化归档      |
| **数字孪生运维**     | 实时接入设备数据，三维数字孪生大屏，智能告警，预测性维护，运维工单闭环 |
| **统一门户与权限**   | 单点登录，动态菜单，角色级权限控制，一个门户访问所有模块     |

---

## 二、各模块详细功能

### M01 – 统一认证与权限管理

**定位**：平台认证中枢，仅提供登录、JWT 签发、用户信息、动态菜单 API，不参与业务数据存储。

#### 功能清单

| 功能模块   | 具体功能                                                     |
| ---------- | ------------------------------------------------------------ |
| 用户认证   | - 用户名/密码登录、JWT 生成与刷新<br>- 登出（Token 黑名单可选）<br>- 支持多因素认证（预留） |
| 用户管理   | - 用户增删改查、启用/禁用<br>- 用户角色分配<br>- 用户信息维护（姓名、手机、邮箱） |
| 角色管理   | - 角色增删改查<br>- 角色权限分配（功能权限、数据权限）       |
| 菜单管理   | - 动态菜单树配置（支持三级）<br>- 菜单项关联 iframeUrl（子应用入口）<br>- 菜单排序、隐藏/显示 |
| 权限点管理 | - 权限点定义（按钮级、API 级）<br>- 角色与权限点绑定         |
| 安全审计   | - 登录日志、操作日志查询与导出（可选）                       |

---

### M02 – 数通网络规划与仿真

**定位**：通信网络规划与覆盖仿真的专业工具，支撑站址选点、容量评估、覆盖优化。

#### 功能清单

| 功能模块           | 具体功能                                                     |
| ------------------ | ------------------------------------------------------------ |
| 基站站址规划       | - 基于 GIS 地图的站址标注与编辑<br>- 自动站址推荐（OR-Tools 算法，考虑覆盖、成本、地形）<br>- 规划方案保存、版本管理 |
| 传播模型与覆盖仿真 | - 支持多种传播模型（Okumura-Hata、COST231 等）<br>- 单站/多站 RSRP、SINR 仿真计算<br>- 覆盖热力图生成与导出 |
| 干扰分析           | - 同频/邻频干扰分析<br>- 干扰矩阵展示与优化建议              |
| 容量评估           | - 话务模型配置<br>- 基站容量预测与扩容建议                   |
| 仿真报告自动生成   | - 一键生成 Word/PDF 格式仿真报告<br>- 包含仿真参数、覆盖图、分析结论 |
| 与 M03 联动        | - 规划站点导出为共享表（shared_station），供 M03 三维设计直接读取 |

---

### M03 – BIM+GIS 三维设计引擎

**定位**：通信基础设施三维数字化设计工具，支持机房、基站、管线的 BIM 建模与空间分析。

#### 功能清单

| 功能模块        | 具体功能                                                     |
| --------------- | ------------------------------------------------------------ |
| GIS 三维场景    | - 加载全球地形/影像/建筑白模<br>- 支持 WMS/WMTS/3D Tiles 等多源数据<br>- 场景快照、标注、标绘 |
| BIM 建模        | - 通信标准化构件库（机柜、天线、走线架、井室等）<br>- 参数化快速建模（修改尺寸、材质、位置）<br>- 模型编辑（移动、旋转、阵列） |
| 模型管理        | - 模型上传（IFC/glTF/SKP 等格式）<br>- 模型轻量化与格式转换<br>- 版本管理、权限控制 |
| 工程设计工具    | - 机房布局设计（自动对齐、间距校验）<br>- 管线综合设计（路由规划、管径校验）<br>- 天线安装设计（方位角、下倾角参数化） |
| 碰撞检测        | - 硬碰撞/软碰撞检测<br>- 碰撞报告自动生成（标注位置、类型、优化建议） |
| 工程量统计      | - 自动统计设备数量、管线长度、材料用量<br>- 导出工程量清单（Excel） |
| 与 M02/M05 联动 | - 读取 M02 规划站点并在三维场景中展示<br>- 竣工模型导出为 3D Tiles 供 M05 加载 |

---

### M04 – 数智化交付与工作流

**定位**：通信工程施工与验收的数字化流程管理，实现“设计即交付、交付即归档”。

#### 功能清单

| 功能模块     | 具体功能                                                     |
| ------------ | ------------------------------------------------------------ |
| 施工流程管理 | - 施工任务分解、派发、进度跟踪<br>- 施工图纸/料单自动生成（基于 M03 设计数据）<br>- 施工日志、现场照片上传 |
| 验收流程管理 | - 验收模板配置（分项验收、竣工验收）<br>- 验收问题记录与整改闭环<br>- 在线签字、电子签章 |
| 工作流引擎   | - 基于 Camunda 的流程设计器<br>- 审批流程、整改流程、变更流程自定义 |
| 交付包管理   | - 自动打包竣工资料（图纸、模型、报告、影像）<br>- 数字化归档至 MinIO |
| 工单管理     | - 接收 M05 告警生成的运维工单<br>- 工单派发、处理、反馈、评价 |
| 外部集成     | - 支持飞书/Excel 导入导出交付清单<br>- 与运营商档案系统对接（预留） |

---

### M05 – 数字孪生与智慧运维

**定位**：基于数字孪生的通信设施智慧运维平台，实时监控、智能告警、预测维护。

#### 功能清单

| 功能模块               | 具体功能                                                     |
| ---------------------- | ------------------------------------------------------------ |
| 数字孪生底座           | - 加载 M03 竣工模型（3D Tiles）<br>- 设备数字镜像构建（属性、文档、历史数据）<br>- 孪生场景漫游、剖切、测量 |
| 设备资产全生命周期     | - 设备台账管理（入库、安装、维修、退役）<br>- 二维码/RFID 扫码盘点<br>- 维保合同、计划管理 |
| 实时监控与告警         | - 对接 EMQX，采集设备 MQTT 数据（温湿度、电压、告警码等）<br>- 可视化监控面板（机房/基站状态）<br>- 告警规则配置（阈值、复合条件）<br>- 告警推送（短信、邮件、企业微信）<br>- 告警闭环（确认→处理→复查） |
| 智能巡检               | - 巡检计划自动生成（周期、路线）<br>- 移动端巡检（扫码、拍照、定位）<br>- 巡检异常自动生成整改工单 |
| 故障自动定位与根因分析 | - 基于设备拓扑与告警关联，快速定位故障点<br>- AI 根因分析（预留，初期规则引擎） |
| 能耗与性能优化         | - 设备能耗实时监控与分析<br>- 空调/电源智能调优策略<br>- 网络性能 KPI 监控（覆盖、干扰、容量） |
| 运维工单               | - 告警/巡检异常自动生成工单（写入 M04 工单表）<br>- 工单处理进度跟踪 |
| 大屏中心               | - 独立可视化大屏（运维总览、告警分布、能耗 TOP N）<br>- 支持大屏拼接、自动轮播 |

---

### M06 – 统一前端门户

**定位**：纯前端“壳工程”，提供统一布局、动态菜单、iframe 集成，不包含业务逻辑。

#### 功能清单

| 功能模块    | 具体功能                                                     |
| ----------- | ------------------------------------------------------------ |
| 登录/登出   | - 调用 M01 登录接口，存储 JWT<br>- 登出清理 token，跳转登录页 |
| 动态菜单    | - 根据用户角色从 M01 获取菜单树<br>- 菜单权限控制（无权限菜单自动隐藏） |
| 布局框架    | - 顶部导航栏（Logo、项目名、全局搜索、通知、个人中心）<br>- 左侧可折叠菜单栏<br>- 右侧 iframe 内容区 |
| iframe 集成 | - 点击菜单，切换 iframe src 加载子应用<br>- 向子应用 postMessage 传递 JWT |
| 全局通知    | - 接收子应用发来的消息（成功/错误/警告）<br>- 统一使用 Element Plus Message 展示 |
| 个人中心    | - 修改个人信息、密码<br>- 查看我的角色、权限、操作日志       |

---

### 大屏中心（独立前端，属于 M05 扩展）

**定位**：面向指挥中心、汇报场景的可视化大屏，集中展示运维关键指标。

#### 功能清单

| 功能模块       | 具体功能                                                     |
| -------------- | ------------------------------------------------------------ |
| 运维总览大屏   | - 全网设备在线率、告警总数、未处理工单数<br>- 区域分布图（GIS 热力） |
| 告警监控大屏   | - 实时告警滚动列表<br>- 告警级别饼图、趋势折线图<br>- 告警 TOP 设备 |
| 能耗分析大屏   | - 基站/PON 口能耗排名<br>- 节能效果对比图<br>- 电费预估      |
| 站址健康度大屏 | - 基于覆盖、干扰、告警、能耗的综合健康度评分<br>- 站址列表下钻分析 |
| 大屏配置工具   | - 拖拽式布局编辑器<br>- 数据源绑定（M05 API）<br>- 全屏/自动轮播 |

## 三、模块间数据流与依赖关系

| 数据流向              | 方式                                                | 说明                      |
| --------------------- | --------------------------------------------------- | ------------------------- |
| M02 → M03             | M02 写入 `m02_plan`，M03 读取                       | 规划站址作为三维设计输入  |
| M03 → M05             | M03 写入 `m03_bim_model`（含 MinIO 路径），M05 读取 | 竣工模型供数字孪生加载    |
| M05 → M04             | M05 直接 `INSERT INTO m04_work_order`               | 告警/巡检异常生成运维工单 |
| M01 → 所有模块        | 各模块前端调用 M01 菜单/用户接口，后端共享 JWT 密钥 | 统一认证与菜单            |
| 所有模块 → shared_ 表 | 只读共享主数据（区域、站点字典）                    | 避免重复维护              |

## 四、部署与访问关系

- **用户访问入口**：M06 门户（`/`）  
- **子应用嵌入**：M02~M05、大屏中心各自独立部署，通过 M06 菜单中配置的 `iframe_url` 加载  
- **后端 API 前缀**：  
  - `/api/m01/*` → M01 服务  
  - `/api/m02/*` → M02 服务  
  - `/api/m03/*` → M03 服务  
  - `/api/m04/*` → M04 服务  
  - `/api/m05/*` → M05 服务  
- **静态资源路径**：`/modules/{m02,m03,m04,m05,screen}/` 映射到各自前端构建产物
