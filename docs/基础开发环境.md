## 五、版本兼容性约束

### Java 与 Spring Boot
- Spring Boot 3.1.x **要求 JDK 17 或更高**。
- 项目 pom.xml 中 java.version=17。

### Node 与 Vite
- Vite 5 需要 Node.js 18+，推荐 20/22 LTS。
- 如果使用 Node 22，Vite 5 完全兼容。

### Cesium 与 Vue
- Cesium ^1.141.0 需要现代浏览器，与 Vue 3 无冲突。
- `vite-plugin-cesium` 需要 Vite 5 和 Cesium 1.141+。

### MySQL
- MySQL 8.0，使用 `mysql:8.0` 镜像。
- 不支持 MySQL 5.7（JSON 功能与空间索引差异）。

## 六、升级策略与检查

1. **依赖升级审批**：任何核心依赖（Spring Boot、Vue、Cesium）的版本升级需团队评审。
2. **版本冻结期**：项目发布前 2 周，停止所有依赖大版本升级。
3. **自动化检查**：建议使用 `renovate` 或 `dependabot` 自动检测依赖更新，但需人工合并。

### 快速验证脚本（Windows PowerShell）

```powershell
# 检查关键工具版本
Write-Host "Java: " -NoNewline; java -version 2>&1 | Select-String "version"
Write-Host "Maven: " -NoNewline; mvn -v | Select-String "Apache Maven"
Write-Host "Node: " -NoNewline; node -v
Write-Host "npm: " -NoNewline; npm -v
Write-Host "MySQL: " -NoNewline; mysql --version
Write-Host "Redis: " -NoNewline; redis-cli --version
Write-Host "Git: " -NoNewline; git --version
Write-Host "Docker: " -NoNewline; docker --version
```

## 附录：推荐环境配置示例

### Windows 开发环境
- JDK 17（与 pom.xml 一致）
- Maven 3.9.16 (阿里云镜像)

- MySQL 8.0.46
