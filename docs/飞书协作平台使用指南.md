## 三、核心功能使用

### 3.1 项目任务表（多维表格）

#### 3.1.1 访问方式

1. 在飞书左侧导航栏点击「更多」
2. 找到并点击「多维表格」
3. 在表格列表中找到 **「通信基建数智化平台任务表」**

#### 3.1.2 表格结构

表格包含5个数据表，对应5个开发模块：

| 数据表名称 | 负责人 | 用途 |
| :--- | :--- | :--- |
| M01-认证中心 | 人员一 | 记录认证中心模块开发任务 |
| M03-BIM+GIS设计 | 人员二 | 记录BIM+GIS模块开发任务 |
| M04-数智化交付 | 人员四 | 记录数智化交付模块开发任务 |
| M05-智慧运维 | 人员三 | 记录智慧运维模块开发任务 |
| 数字大屏 | 人员五 | 记录数字大屏模块开发任务 |

#### 3.1.3 字段说明

每个数据表包含以下字段：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| 序号 | 数字 | 自动编号 |
| 功能模块 | 文本 | 功能分类名称 |
| 任务描述 | 文本 | 详细任务说明 |
| 负责人 | 人员 | 任务负责人 |
| 完成状态 | 单选 | 未开始/进行中/已完成 |
| 优先级 | 单选 | 高/中/低 |
| 截止日期 | 日期 | 任务截止时间 |
| 备注 | 文本 | 其他说明信息 |

#### 3.1.4 日常操作指南

**查看任务**
- 点击对应的数据表标签（如「M04-数智化交付」）
- 浏览自己负责模块的所有任务

**修改任务**
1. 找到需要修改的任务行
2. 点击对应单元格进行编辑
3. 修改完成后自动保存

**添加新任务**
1. 点击表格右下角的「+」按钮
2. 填写任务信息
3. 选择负责人和优先级
4. 设置截止日期

**更新状态**
1. 找到自己负责的任务
2. 点击「完成状态」列的下拉框
3. 选择「进行中」或「已完成」

#### 3.1.5 注意事项

- ✅ 每周至少更新一次任务状态
- ✅ 任务完成后及时标记为「已完成」
- ✅ 遇到问题及时在备注栏说明
- ❌ 不要修改其他人员负责的任务（除非有明确授权）

### 3.2 问题收集表（多维表格）

#### 3.2.1 访问方式

1. 在多维表格列表中找到 **「项目问题收集表」**

#### 3.2.2 字段说明

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| 序号 | 数字 | 自动编号 |
| 问题描述 | 文本 | 详细描述遇到的问题 |
| 问题类型 | 单选 | 代码问题/环境问题/需求问题/其他 |
| 严重程度 | 单选 | 严重/中等/轻微 |
| 发现人 | 人员 | 问题发现者 |
| 发现日期 | 日期 | 自动记录 |
| 状态 | 单选 | 待处理/处理中/已解决/已关闭 |
| 处理人 | 人员 | 负责处理的人 |
| 备注 | 文本 | 处理进展说明 |

#### 3.2.3 使用场景

**记录问题**
- 开发过程中遇到任何问题都应记录
- 包括代码bug、环境配置问题、需求不明确等

**问题处理**
- 项目经理会定期查看并分配处理人
- 处理人应及时更新处理进展

**例会使用**
- 每周例会前整理问题收集表
- 重点讨论未解决的严重问题

## 四、协作规范

### 4.1 更新频率

| 内容 | 更新频率 | 说明 |
| :--- | :--- | :--- |
| 任务状态 | 每周 | 每周更新任务状态 |
| 问题记录 | 即时 | 发现问题立即记录 |
| 周进度报告 | 每周五 | 总结本周工作 |

### 4.2 沟通渠道

| 场景 | 渠道 | 说明 |
| :--- | :--- | :--- |
| 紧急问题 | 飞书即时消息 | 直接@相关人员 |
| 技术讨论 | 飞书群聊 | 在项目群中讨论 |
| 文档协作 | 飞书文档 | 共同编辑文档 |
| 会议沟通 | 飞书会议 | 线上会议讨论 |

### 4.3 文档管理

- 项目相关文档统一存放在飞书文档空间
- 文档命名规范：`[模块]-[文档名称]-[版本]`
- 重要文档设置权限，确保安全
