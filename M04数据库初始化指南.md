# M04 模块数据库初始化指南

## 问题说明
你遇到的问题是：新建项目后页面显示「No Data」，搜索也找不到数据。这是因为数据库表还没有创建，或者没有测试数据。

## 解决方案

### 方案一：使用 MySQL 命令行工具（推荐）

#### 1. 打开命令提示符或 PowerShell

#### 2. 进入 MySQL 并执行脚本
```bash
mysql -u root -p < scripts/init-m04-simple.sql
```
按提示输入密码（默认为 `123456`）

### 方案二：使用 MySQL Workbench 或其他图形化工具

1. 打开 MySQL Workbench
2. 连接到本地 MySQL 服务器
3. 点击「File」→「Open SQL Script」
4. 选择 `scripts/init-m04-simple.sql` 文件
5. 点击闪电图标（执行）

### 方案三：如果 MySQL 不在 PATH 中

你需要先找到 MySQL 的安装路径，通常在：
- `C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe`
- 或者 `C:\Program Files\MySQL\MySQL Server 5.7\bin\mysql.exe`

然后执行：
```bash
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p < scripts/init-m04-simple.sql
```

## 验证数据库是否成功初始化

执行完脚本后，你可以：

1. 在数据库中查看表是否创建：
```sql
USE comm_platform;
SHOW TABLES;
-- 应该能看到 m04_project 和 m04_work_order 等表
```

2. 查看示例数据：
```sql
SELECT * FROM m04_project;
-- 应该能看到 3 条示例项目数据
SELECT * FROM m04_work_order;
-- 应该能看到 3 条示例工单数据
```

## 测试流程

数据库初始化完成后：

1. 刷新浏览器页面（http://localhost:5173/）
2. 点击侧边栏「数智化交付」→「文档管理」
3. 应该能看到 3 条示例项目数据了！
4. 可以尝试新增、编辑、删除项目功能

## 如果仍然有问题

### 检查后端服务是否正常
打开浏览器访问：
```
http://localhost:8084/api/m04/project?pageNum=1&pageSize=10
```
如果返回 JSON 数据，说明后端正常。

### 检查前端代理配置
确认 `packages/m06-portal/vite.config.js` 中配置了正确的代理：
```javascript
proxy: {
  '/api/m04': {
    target: 'http://localhost:8084',
    changeOrigin: true,
    rewrite: (path) => path
  }
}
```

### 重启服务
如果修改了配置，记得重启前端和后端服务。

## 数据库连接信息

根据 `application.yml` 配置：
- 数据库地址：localhost:3306
- 数据库名：comm_platform
- 用户名：root
- 密码：123456
